このチュートリアルには2つの目的があります。プログラミングとプログラミング言語の基本的な概念を教えること、そして、それらが Python 3 言語でどのように実装されるかを示すことです。そのために、科学者が遭遇しうる2つの典型的なユースケースを取り上げます。
- **誰かのプロトコルを実行する**。この状況では、あなたの問題に対する確立された解法があり、それを解く高品質なソフトウェアがすでに存在します。例として、*発現差問題（differential expression problem）* を取り上げ、実験群と対照群それぞれ5反復のデータに対して t 検定を実行するために Python を使います。

このチュートリアルは、プログラミングや計算機科学の正式な前提知識を必要としませんが、分子生物学にはなじみがあることを前提とします。論理、集合論、線形代数、統計といった数学的なトピックにいくらか親しんでいると役立ちますが、必須ではありません。

## 逆相補鎖問題
**定義**。次の DNA 配列を考えます。
```
5'-GACTACAAAGACGATGACGATAAA-3'
   ||||||||||||||||||||||||
3'-CTGATGTTTCTGCTACTGCTATTT-5'
```
この表記は、分子生物学者にはおなじみの DNA のいくつかの性質を示しています。第一に、*相補的（complementary）* です。一方の鎖のすべての `A` は他方の `T` と対合し、すべての `G` は `C` と対合します。第二に、*逆平行（antiparallel）* です。つまり、各鎖の 5' から 3' への向きが互いに逆方向です。これらの性質を合わせると、反対側の鎖の配列は推測できるため、DNA は1本の文字列 `GACTACAAAGACGATGACGATAAA` として表せます。しかし、相補配列を得ることが重要な場合もあります。たとえば PCR のプライマー設計では、実際に増幅したい配列に相補的なプライマーを注文する必要があります。

どうすればよいか少し考えてみてください。配列が1つか2つだけなら手作業でできるかもしれません。しかし、100万個のプライマーが必要だとしたら…？ それを代わりにやってくれるプログラムが欲しくなります。もちろん、まさにこの問題のために作られたプログラムはすでに存在しますが[^1]、自分で書くのに十分なほど簡単な演習なので、ここでは自作します。

*逆相補鎖問題：*
*DNA 配列 x が与えられたとき、その逆相補鎖 DNA 配列 y は何か？*
「DNA 配列」と明記することで、ある配列が DNA と RNA の両方の相補鎖を持つという曖昧さを避けます。さらに、解は $y=f(x)$ の形の関数であることを要件とします。

**解法**。問題を定義できたので、解法の実装に取りかかれます。ここで、問題（problem）、解法（solution）、実装（implementation）の区別に注目する価値があります。問題の記述では、コードを一切書かずに、関数の入力と出力が何であるべきかを数学的に定義しました。さらに、問題を解くために必要な具体的な手順を定義する *解法*、いわゆる *疑似コード（pseudocode）* を、*やはりコードを書かずに* 指定することもできます。最後に、指定された処理を実際に実行する、特定のプログラミング言語で書かれたコードによる *実装* があります。新しいプログラミング言語に不慣れな場合は、(1) コードが正確に何をするか、(2) どのようにそれを行うか、を明確にしておくと役立ちます。これらは、(2) で指定された処理を実行する (3) 特定のプログラミング言語のコードとは、ほぼ独立しています。

たとえば逆相補鎖問題では、上で問題をすでに定義しました。逆相補鎖問題を解く疑似コードは、次のように書くこともできます。
```
配列 x が与えられたとき：
x を反転する；
x を相補にする；
完了
```
あるいは同等に、
```
配列 x が与えられたとき：
x を相補にする；
x を反転する；
完了
```
ある問題に対して正しい解が複数存在しうることに注意してください。最後に、この疑似コードを実装するために必要な Python 言語の概念を、チュートリアル 1.1 と 1.2 で扱います。

## 発現差問題
**定義**。次に、非常によくある次のシナリオを考えます。あなたは科学者で、患者コホートと対照コホート、あるいは細胞株モデルの処理群と対照群といった、2 組の観測を持っているとします。群の間で何が「違う」のかを知りたいので、RNA-seq のようなハイスループットのオミクス実験を行います。データ処理の後、いまや *n* 個のサンプルそれぞれについて *k* 個の遺伝子発現の測定値を持つ巨大な表が手元にあります。同様のデータは、タンパク質質量分析、ATAC-seq、ChIP-seq、シングルセル、空間トランスクリプトミクスといった他のハイスループット実験でも得られます。

*発現差問題：*
*n 個の観測にわたる k 個の特徴量からなる大きな表と、各 n に関連づけられた注釈 a が与えられたとき、a が指定するサンプル群に関して、k のうちどの特徴量が異なるか？*

あるいは、より口語的に言えば、

*どの遺伝子が発現差を示すか？*

**解法**。これは非常によくある問いで、答えるための方法は数多くあります。これらの方法は細部では異なりますが、大まかな考え方は同じです。
```
k の各遺伝子 g について：
a に関する g の統計モデルを構築する；
有意に異なる遺伝子を特定する；
完了
```

RNA-seq については、2025 年時点での現在の標準は DESeq2[^3] と edgeR[^4] というソフトウェアツールです。どちらのツールも一般化線形モデル（GLM）を用いてデータを当てはめ、RNA-seq データの分布について仮定を置きますが、これらは後のチュートリアルで扱います。ここでの目的のためには、チュートリアル 1.0 で t 検定を行い、*g* の分布が群間で有意に異なるかを判定します。その後、チュートリアル 1.3 で、*k* のすべての遺伝子について t 検定を行い、約 20,000 回の検定を行ったことを考慮するために *多重仮説補正（multiple hypothesis correction）* を実施します。

### p 値と多重仮説補正についての注記

この節では *多重仮説補正* を紹介します。すでに馴染みがあれば、読み飛ばしてチュートリアルに進んで構いません。そうでなければ、続けて読んでください。

まず、有名な「p 値」から始めましょう。それが何で、どう使うかについて、すでに何らかの直観をお持ちかもしれません。私の直観は、「$p < 0.05$ なら有意、そうでなければ非有意」といったところです。しかし、*p* はどのように決まり、何を意味するのでしょうか？

ほとんどの統計検定は、「$a$ と $g$ の間に関係はない」といった趣旨の *帰無仮説（null hypothesis）* $H_0$ から始まります。$H_0$ が真であると仮定すると（通常は他の仮定も置きます）、データがどのように生成されるかの統計モデル、すなわち *帰無分布（null distribution）* を構築できます。*p 値* は、与えられたデータで観測された結果と少なくとも同じくらい極端な結果を、帰無分布が生成する確率として定義されます。たとえば、チュートリアル 1.0 で行う2標本 t 検定は、データが同じ平均を持つ2つの t 分布から抽出されたと仮定します。p 値が 0.05 であるとは、それらの分布が我々の観測と同じくらい異常なデータを生成する確率が 20 分の 1 である、ということです。これは起こりにくいので、その場合には、2 群の平均が等しいという仮定 $H_0$ を棄却するのに十分な証拠があることになります。ただし、$p<0.05$ という閾値は、*偽陽性（false positives）* のリスクをいくらか受け入れています。つまり、真の差が存在しなくても有意な結果が得られる確率が 5% あります。

次に、同じサンプルを同じ条件で生物学的に反復し、無作為に2群に分けて RNA-seq を行った場合の結果を考えてみましょう。生物学的な差がないので帰無仮説は真であり、したがって有意な結果はすべて偽陽性のはずです。20,000 個のタンパク質コード遺伝子すべてについて t 検定を行ったとすると、$20,000 \times 0.05 = 1,000$ 個の有意な結果が、*群間に真の差が存在しないにもかかわらず* 期待されます。当然ながら、1,000 個の偽陽性の結果（さらに悪いことに、1,000 件の偽陽性結果の詳細な検討）を含む論文を発表することは、膨大な時間の浪費になります。

この問題を解決する方法として認められているものが2つあります。1つは、20,000 回の検定を行わないことです。あらかじめ1つの遺伝子だけに関心があると分かっているなら、残りの 19,999 個を検定しないよう、その1つの遺伝子を調べるように実験を設計するのが妥当です（おそらく RNA-seq は使いません）[^5]。

もう1つの方法は、$p<0.05$ の閾値（通常は「調整済み *p*」または *q 値* として報告されます）を調整して、期待される偽陽性の数を制限することです。たとえば上記の RNA-seq 実験では、より厳しい $p=\frac{0.05}{20,000}=2.5 \times 10^{-6}$ という閾値を設定し、20,000 個の仮説の *いずれか* で偽陽性が生じる累積確率を 5% にすることができます。これは *ボンフェローニ補正（Bonferroni correction）* として知られています。もう1つは、期待される偽陽性が結果の *q* を超えないように *q 値* を設定することです。つまり、$q<0.1$ という閾値は、結果の 10% が偽陽性と期待されることを示します。これを行う数学は範囲外ですが、標準的な方法は *Benjamini-Hochberg FDR 補正* です。BH 補正は RNA-seq の発現差解析の標準であり、チュートリアル 1.3 で多重仮説を検定する際に使用します。
